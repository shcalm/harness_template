## 4. 需求清单

| REQ-ID | 优先级 | 状态 | 需求描述 | 关联流程 |
| :--- | :--- | :--- | :--- | :--- |
| REQ-001 | P0 | `[草案 / 确认 / 变更中 / 完成]` | `[用户或系统必须具备的能力]` | FLOW-001 |

优先级定义：
* P0：它是交付核心流程的必要条件。
* P1：影响主要体验或稳定性，但不阻断最小闭环。
* P2：增强项或后续优化。

## 5. 端到端验收标准

| AC-ID | 关联 REQ | 场景 | 输入 | 步骤 | 预期结果 | 证据类型 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| AC-001 | REQ-001 | `[业务场景]` | `[具体输入数据]` | `1. ... 2. ...` | `[API 返回 / UI 状态 / DB 变更 / 文件产物 / 日志特征]` | `[测试命令 / 日志 / 截图 / 产物路径]` |

验收标准要求：
* 每条 `AC-ID` 只验证一个业务结论。
* 预期结果必须能被 `TEST_REPORT.md` 记录为证据。
* 如果必须人工观察，写明观察窗口、观察对象和通过条件。
